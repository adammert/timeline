# CSS Style Guide

## Architecture
- Use **CSS Variables** for colors, spacing, and reusable values.
- Maintain a clear distinction between light and dark themes.

## Naming
- Use **BEM-like** naming or clear semantic classes.
- Prefix state classes with `is-` (e.g., `is-active`, `is-critical`).

## Formatting
- Group related properties together (e.g., layout, then box model, then typography).
- Use 2-space indentation.
- Always include a semicolon.

## Best Practices
- Avoid deeply nested selectors.
- Use `rem` or `em` for font sizes instead of `px`.
