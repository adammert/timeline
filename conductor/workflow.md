# Project Workflow - Conductor

## General Workflow
This project follows the Conductor methodology for development.

## Development Rules
1. **Task-Driven Development:** All work is organized into tracks, phases, and tasks.
2. **Commit Policy:** Commit changes after the successful completion of every task.
3. **Commit Messages:** Use clear, concise commit messages.
4. **Task Summaries:** Use Git Notes to record a summary of the changes for each task.
5. **Testing:** Maintain a minimum of 80% code test coverage.
6. **Documentation:** Keep the project documentation (product guide, tech stack, etc.) updated as the project evolves.

## Phase Completion Protocol
At the end of each phase, a manual verification task is required:
- [ ] Task: Conductor - User Manual Verification '<Phase Name>'
  - Protocol: The developer must manually verify that all features in this phase meet the requirements and design guidelines before proceeding to the next phase.
